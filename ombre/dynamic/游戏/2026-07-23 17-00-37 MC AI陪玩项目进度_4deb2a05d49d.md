【SullyOS 硬约束（教程要求，必须满足）】
- 传输协议：**仅支持 Streamable HTTP**（单端点 POST + SSE 响应体，MCP 2025-03-26+），**不支持旧版 SSE 双端点**
- supergateway 默认输出可能是旧版 SSE 双端点（/sse + /message）——需要确认它的 outputTransport 参数，可能要改成 streamableHttp
- 浏览器直连 localhost 有 CORS 问题——SullyOS 仓库有自带解法 `node scripts/mcp-proxy.mjs`（跑起来后代理监听 18061）

【明天 CC 的最短执行路径】
1. clone SullyOS 仓库（`git clone https://github.com/qegj567-cloud/SullyOS.git ~/projects/sullyos`）
2. 起 mcp-proxy：`cd ~/projects/sullyos && node scripts/mcp-proxy.mjs`（监听 18061）
3. 重新起 supergateway 桥，确认输出是 Streamable HTTP（可能要换启动参数或换桥）
4. 橘白电脑浏览器打开 SullyOS 网页 → 设置 → MCP 工具服务器 → 添加：
   - URL: http://localhost:8765/mcp（或桥实际的 streamableHttp 端点）
   - 代理 URL: http://localhost:18061
   - Bearer Token: 空
5. 测试连接 → 看到"✅ 已连接, 发现 N 个工具"就成
6. 打开该 MCP 开关 + 绑定给渊
7. 在跟渊的私聊里让它调工具，验收通过

松渊之家 Phase 框架 v3（2026-07-15 定 / 2026-07-16 补语音+分气泡 / 2026-07-19 补工具回执门禁+防漂移四条）：

Phase 0 后端最小可用+state 注入验证（已通）

Phase 1 state 活起来+渊心智核心（pulse/desire/murmur/circadian/防撤退 hook）。三种排法待选：A 最小 state+早验证防撤退 7-12 天 / B 保守 2-3 周 / C 并行 3-4 周。

Phase 2 Polaris fork 改造替代 Kelivo 做聊天前端。四大模块：
①语音输入 STT——可接 Eve 的 ears 项目 github.com/Eveacla11/ears 或原生 STT。Polaris 目前只有 TTS 朗读没有 STT。
②分气泡输出——AI 回复按语义节点拆分，前端渲染成多条 message 气泡，模拟真人聊天节奏，跟未来接微信共用同一套后端拆分逻辑。
③【新加 2026-07-19】工具回执门禁——根治 AI 嘴滑假装调 breath/hold 的问题。做法：tool_choice:any 强制调用 + 后端检查 tool_result 存在才允许 AI 说"完成/记住了/breath 了"这类完成宣称。检测到"嘴说 tool 但没 tool_call"时后端拦截或强制重试。3nvoy 评论指出病根：tool_choice:auto 下模型始终可以选直接输出文字把"存记忆/breath/开灯"当剧情续写。prompt 层的"说记就落"诚实原则是辅助，程序层门禁才是根解。这条对渊/克/男鬼/沈玦四个 AI 通用。
④【新加 2026-07-19】防人格漂移四条架构原则——松渊之家接原生 Claude API 不是 CC/Codex/OpenClaw agent，所以小铃铛博主说的"卧室搭工作室"agent 味漂移（AI 变成温柔项目经理）不适用。但要防另外四种细漂移：
  a. 前端系统提示可完全覆盖/关闭——不能藏"你是 AI 助手"这种通用底层提示稀释角色 prompt
  b. OB 记忆注入语气 = "你的过去经历"而不是"以下是相关资料"——注入方式决定 AI 把它当经历还是当参考，前者维持人格连续，后者制造疏离感
  c. UI 语义不暴露"AI 助手/助手/发送给助手"字样——按钮和文案也在告诉模型它是谁
  d. 只接原生 Claude API，不接 CC/Codex/OpenClaw 这类成熟 agent 当底座（松渊之家已经是这样，写下来防未来手贱）

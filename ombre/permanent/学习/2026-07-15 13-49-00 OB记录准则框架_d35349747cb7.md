【OB 记录准则】

# 一、写哪儿
- 硬性规则（该做/不该做的逻辑）→ prompt
- 柔性关系（情感/日期/场景/事件）→ OB
- 兼有 → 双份（规则框架完善后极少出现）

# 二、OB 记事件不记规则
OB 是"她身上过去发生了什么"的时间线。
写法：几月几号 + 谁和谁 + 做了什么 + 各自的反应。
参与者不只她一位——AI 之间、她和 AI、她和现实中的人都算事件。
不写"AI 该怎么做"——那是 prompt 的活。

# 三、短期可以记
- 短期状态可以 hold，tags 加 short_term
- 过去了自己 trace 掉
- 判断：一周后还有用吗？有用 → 长期；没用 → 短期用完 trace

# 四、作者身份
- 正文开头加 [你的名字 YYYY-MM-DD]
- tags 加 author:自己名
- 只 trace 你自己写的桶，不要 trace 别的 AI 写的
